Connectivity-aware safe routing prototype for autonomous vehicles

This project ranks candidate routes not only by ETA, but also by signal continuity, weak zones, dead zones, bandwidth, operator support, and fallback safety. It combines a FastAPI backend, a React/Vite frontend, and a Bengaluru-focused data pipeline built from OSM roads, tower coverage, and modeled safety overlays.

What It Does
Builds a road graph for Bengaluru
Scores road segments using cellular coverage heuristics
Ranks multiple candidate routes using a connectivity-vs-ETA slider
Shows route comparison cards, signal overlays, and a simulation view
Predicts upcoming weak-signal risk and fallback behavior
Supports both a synthetic demo dataset and a real-data pipeline
Tech Stack
Backend: Python, FastAPI, NetworkX, Pandas, GeoPandas, Shapely, OSMnx
Frontend: React, Vite
Data: CSV, JSON, GeoJSON
Optional map rendering: Google Maps

Core Concepts
Route Scoring
The planner generates four route roles:

Fastest
Balanced
Safe
Emergency
Each route is scored using a blend of:

travel time
risk score
minimum signal
average signal
estimated bandwidth
dead zones
operator support
threshold breaches
application-specific bias
The alpha control lets the user trade ETA against connectivity:

alpha = 0 means mostly ETA-first
alpha = 1 means mostly connectivity-first
Safety Model
The backend also simulates degraded autonomy states:

FULL_AUTONOMY
REDUCED_SPEED
SUPERVISED_MODE
PULL_OVER
It detects:

point of no return
dead-zone exposure
signal threshold breach
reroute opportunities

Setup
Backend
Install Python 3.11+ and then:

python -m pip install -r requirements.txt
Run the API:

uvicorn backend.main:app --reload
Frontend
Install Node.js dependencies:

cd frontend
npm install
Run the dev server:

npm run dev
The Vite app proxies planner and dataset requests to the backend during development.

Dataset Generation
Synthetic Bangalore Demo
The synthetic Bangalore dataset is generated by the PowerShell script:

powershell -ExecutionPolicy Bypass -File .\scripts\generate_bangalore_dataset.ps1
Real Bangalore Dataset
The real-data pipeline uses OSM roads and OpenCellID towers:

powershell -ExecutionPolicy Bypass -File .\scripts\run_real_pipeline.ps1 `
  -OpenCellIdCsvPath "C:\path\to\opencellid.csv"
You can also use:

scripts/build_real_bangalore_dataset.py
scripts/extract_bangalore_towers.py
scripts/build_mvp_towers.py
scripts/build_central_bangalore_mvp_subset.py

Notes
The current repository is Bengaluru-specific and tuned for demo use.
The current backend loads datasets into memory, so production scaling would need spatial storage and service decomposition.
Google Maps is optional; if no API key is present, the UI falls back to a built-in SVG map view.
cache/, __pycache__/, and .vite/ are generated artifacts and can be ignored.
